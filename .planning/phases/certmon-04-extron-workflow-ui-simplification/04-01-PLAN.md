# Phase 04 Plan 01: Device-first Extron Local CA and Upload workflow

<objective>
Restructure CertMon's UI so scanned devices are the starting point, Extron-compatible Local CA device-certificate preparation happens from Devices, Upload shows one primary prepared-device Toolbelt list, Local CA becomes root-management-only, and manual upload is retained only as a collapsed fallback.
</objective>

<must_haves>
## Truths

- D-01: The first tab becomes `Devices` and shows scanned devices as the primary starting point.
- D-02: Extron/Local CA/Upload preparation actions appear only where applicable; non-Extron scanned devices remain visible without the Extron Toolbelt path.
- D-03: Missing Local CA device certificates open a review dialog for CN, SAN, and certificate profile before creation.
- D-04: Existing Extron-compatible Local CA certificates show a clear `Local CA ready` status.
- D-05: Clicking `Local CA ready` first shows existing certificate details.
- D-06: The existing-certificate dialog primary action is `Use existing`, with a secondary issue/replace path and `Cancel`.
- D-07: Recognized Extron devices default to `Extron compatible (RSA)`.
- D-08: After `Create & add to Upload`, CertMon stays on Devices and shows feedback with a path to Upload.
- D-09: Normal device-certificate UI belongs in Devices, not Local CA.
- D-10: Upload primary content is the central prepared-device list for Toolbelt dry-run/upload.
- D-11: Upload always has an `Add device` button that jumps to Devices.
- D-12: Manual upload is collapsed at the bottom of Upload.
- D-13: Remove from Upload requires explicit confirmation and deletes the associated Local CA device certificate.
- D-14: Removed devices disappear from Upload but remain visible in Devices when still scanned.
- D-15: Local CA tab is limited to root CA creation/status, Windows trust check/install, and root certificate download.
- D-16: Device certificate creation/management is removed from Local CA.
</must_haves>
