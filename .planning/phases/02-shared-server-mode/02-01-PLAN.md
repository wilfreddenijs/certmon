# Shared Server Mode and Team Local CA

## Goal

Turn CertMon from a trusted single-user desktop/local service into a safe shared LAN service where multiple operators can sign in, use the same CertMon Local CA for local devices, and manage renewals/deployments without exposing private key material accidentally.

## Primary Outcome

A first supported server-mode MVP with local users, roles, sessions, CSRF protection, user-aware audit logs, guarded private-key export, shared Local CA trust bundle export, and clear operator documentation.

## Non-Goals

- No public internet exposure.
- No unattended scheduled renewal service.
- No full enterprise SSO requirement for MVP.
- No automatic CA-root deployment to all client machines.
- No HA/multi-node clustering.
- No browser delivery of deployment private keys beyond explicit manual export.

## Delivery Slices

1. Server-mode safety gate.
2. Identity and sessions.
3. Roles and permissions.
4. Audit and accountability.
5. Shared Local CA operations.
6. Team UAT and packaging.

## Suggested Roles

- Viewer: view inventory, certificate status, public certificate artifacts.
- Operator: start renewals, complete manual DNS, deploy certs to configured devices.
- CA Admin: manage Local CA, issue Local CA certs, export public CA trust bundle.
- Security Admin: explicit private-key export/download and recovery package operations.
- Admin: manage users, roles, server settings, and audit review.

MVP may combine CA Admin and Security Admin into Admin if the implementation gets too large, but private-key export must remain separately permissioned in code.
